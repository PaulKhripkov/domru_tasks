Описание
========

Выполненные задания расположены в файлах с соответствующими названиями task1.sql, task2.sql и т.д. Задания 1 и 1а - в одном файле task1.sql


## Замечания к заданию 1а

- В задании нет явного указания, включать ли старшего в иерархии сотрудника в выборку. Этот сотрудник исключён.


## Замечания к заданию 2

- Опечатка в задании. Год даты операций в исходной таблице и в примере не совпадают.


## Замечания к заданию 3

- Выход за нижнюю границу дат обрабатывается путём игнорирования дат, меньших, чем минимальная дата в таблице. Можно заменить inner join на left join, а null заменять на 0 в результате. Тогда на все даты, меньшие минимальных, получим сальдо, равное нулю (что логично).
- Для правильной работы запроса, необходимо, чтобы таблица jt$calendar была корректно заполнена на соответствующие диапазоны дат.
- В задании имеется опечатка по тестовому периоду (10.12.2008 – 20.01.2008).


## Замечания к заданию 4

- Сложность задания заявлена как низкая, потому реализован только базовый функционал, удовлетворяющий условиям задания.
- Для использования в продакшене нужно добавить ряд проверок и ограничений, а также описать поведение при передаче некорректных данных или данных, для которых выборка будет пустой. Например, можно ли вручную задавать дату найма повара, могут ли быть блюда с одинаковым названием, может ли повар изучить одно блюдо несколько раз и т.п.
- Возврат из функции списка в формате varchar2 - не лучшее решение, т.к. размер переменной может быть превышен.
